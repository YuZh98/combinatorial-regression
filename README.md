# JASA Reproducibility Repository

This repository contains code and data to reproduce the results in the paper.

All commands must be run from the repository root.

Generated outputs are written to:

```
results/
```

---

# 🚀 Quick Start

### 1️⃣ Smoke test (sanity check)

```bash
make smoke
```

Runs a lightweight simulation and data analysis to verify everything works.

---

### 2️⃣ Main simulation study (MH-within-Gibbs)

```bash
make full
```

- Uses exponential kernel for $u^{(i)}$ (main paper default)
- The default setting does not run full grid of simulation settings in the paper. To run your custom settings, there are two ways. First, you can change the following enviroment variables: 
    
    ```JASA_D_LIST```: a list of values of $d$ to run;

    ```JASA_M_LIST```: a list of values of $m$ to run;

    ```JASA_N_ITER```: the number of iterations;

    ```JASA_N_WARMUP```: the number of warmup steps;

    ```JASA_N_REP```: the number of experiment repetition for each setting;

    ```JASA_METHOD```: the kernel assigned to $u^{(i)}$ (default is "exponential halfgaussian"; however, halfgaussian will not run unless JASA_RUN_TAG is set as kernel_compare).
Another way to use custom settings is to directly change config from line 78 to line 98 in R/simulations/mh_within_gibbs/Production_Run.R

- Outputs saved under: (default is 1000)
  ```
  results/runs/mh_within_gibbs/
  ```
- All quantities in Table 2 (Root mean squared error (RMSE) for estimating β using posterior mean under different dimensionality $(d, m)$) can be generated by running ```make full``` with corresponding settings.

---

### 3️⃣ Kernel comparison (supplementary)

```bash
make kernel_compare
```

- Runs both:
  - exponential
  - half_gaussian
- Uses lighter iteration defaults
- Outputs saved under:
  ```
  results/runs/mh_within_gibbs/kernel_compare/
  ```

---

### 4️⃣ Data analysis (waterfowl matching)

Full model:

```bash
make duck_full
```

Reduced model:

```bash
make duck_reduced
```


---

# ⚙️ Custom Settings

All R scripts support environment-variable overrides.

Example:

```bash
JASA_N_ITER=20000 make full
```

Kernel override:

```bash
JASA_METHODS=exponential,half_gaussian make full
```
